Summoner's War Data Logs

Here you will find a centralized statistical resource for Summoner's War's random drops and summons. The goal is to get definitive answers to common questions like:

  • How often does X drop from Y dungeon?
  • Why do I always seem to summon golems?
  • Which scenario or dungeon is best for farming XP/Mana/whatever?
  • Is timegate real?
  • Is Com2US changing drop/summon rates?

The reports are always a work in progress. If you have any suggestions, feel free to communicate it via the feedback section or as an issue on the Github project page. Reports are cached for 60 minutes at a time, so aggregate logs are not immediately added.

Upcoming features:

  • Individualized reports. Logs and statistics associated with your SWARFARM account. These will update in real-time, unlike the public aggregate reports.
  • Timegate charts - aka frequency of natural 4 and 5 summons throughout the day.
  • Detailed rune drop statistics for Giant's Keep, Dragon's Lair, and Necropolis.
  • Raid drops
  • Wish results

How to contribute

  1. Download the latest package from the SWProxy-plugins project on Github.
  2. Make sure you have a SwarfarmLogger.py plugin in the SWProxy plugins directory.
  3. Play the game while connected to the proxy.

What is collected?

Summons

  • Summoning Method (which scroll you used)
  • Monster Summoned (level, grade, awakened/unawakened)

Dungeon Runs

  • Dungeon
  • Stage or Floor
  • Difficulty (for scenarios)
  • Mana, Crystal, and Energy drops
  • Reward chest contents

In addition, for every log we record a unique ID assigned by Com2US, which allows me to associate the data with your SWARFARM profile if you have one. All publicly accessible data is anonymized and aggregated, so this will never be exposed to other users. Other data logged is the game server you're playing on and a timestamp. If you have any other concerns, feel free to browse the code in the plugin or in the swarfarm_io github project to see what exactly is done with the game data.
